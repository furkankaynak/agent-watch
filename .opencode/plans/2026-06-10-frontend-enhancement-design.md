# Frontend Enhancement: Skills, Rules & Agent Statistics

> **Goal:** Show which skills and rules each agent loaded, enrich agent nodes with inline statistics badges, expand InspectorPanel with tool/file/shell/MCP details, and format new event types in EventFeed.

**Architecture:** Server-side chip generation from `beforeReadFile` hook attachments populates the existing `agent_chips` table. Client-side reducer tracks per-agent counters for tools, files, shell commands, MCP calls, and subagents. InspectorPanel reads from the live event buffer (no new API). All changes are additive — existing components and data flow preserved.

**Tech Stack:** React (with ReactFlow canvas), TypeScript, CSS (BEM), better-sqlite3 (server-side only)

---

## Section 1: Skills & Rules Chip Generation (Server-side)

**Problem:** `agent_chips` table is empty — `skill_read`, `rule_read` events are never generated by any hook, so InspectorPanel shows no skills/rules.

**Solution:** The `beforeReadFile` hook carries `attachments[]` with `type: "rule"` entries. Extract rule names in the hook handler, generate chip events server-side.

### Files

- **Modify:** `cursor-plugin/hooks/ingest.cjs` — `handleBeforeReadFile`
- **Modify:** `server/eventProcessor.ts` — add `case "file_read"` to `processEvent`

### Changes

**ingest.cjs — `handleBeforeReadFile`:**
Add two fields back to the handler:
```javascript
const rules = attachments
  .filter(a => a.type === 'rule')
  .map(a => path.basename(a.file_path || '').replace(/\.mdc$/, ''));
const skillPaths = attachments
  .filter(a => a.type === 'file' && (a.file_path || '').includes('/skills/'))
  .map(a => path.basename(path.dirname(a.file_path || '')));

fields.attachment_rules = rules.length > 0 ? JSON.stringify(rules) : null;
fields.attachment_skills = skillPaths.length > 0 ? JSON.stringify(skillPaths) : null;
```

Also detect skill reads from the read file_path itself:
```javascript
const isSkillFile = (payload.file_path || '').includes('/skills/') || (payload.file_path || '').endsWith('SKILL.md');
if (isSkillFile) {
  const skillName = path.basename(path.dirname(payload.file_path || ''));
  fields.attachment_skills = JSON.stringify([skillName]);
}
```

**eventProcessor.ts — `processEvent`:**
Add `case "file_read"` to the switch statement:
```typescript
case "file_read":
  handleFileRead(db, event);
  break;
```

New `handleFileRead` function:
```typescript
function handleFileRead(db: Database.Database, event: LogEvent): void {
  const bound = bindConversation(event, db);
  if (!bound.agentId) return;

  const rules = safeParseArray(event.fields.attachment_rules);
  for (const rule of rules) {
    db.prepare("INSERT OR IGNORE INTO agent_chips (agent_id, chip_type, chip_value, seen_at) VALUES (?, ?, ?, ?)")
      .run(bound.agentId, "rule", rule, event.timestamp);
  }

  const skills = safeParseArray(event.fields.attachment_skills);
  for (const skill of skills) {
    db.prepare("INSERT OR IGNORE INTO agent_chips (agent_id, chip_type, chip_value, seen_at) VALUES (?, ?, ?, ?)")
      .run(bound.agentId, "skill", skill, event.timestamp);
  }

  touchAgent(db, bound.agentId, event.timestamp);
}

function safeParseArray(raw: string | undefined): string[] {
  if (!raw) return [];
  try {
    const parsed = JSON.parse(raw);
    return Array.isArray(parsed) ? parsed : [];
  } catch { return []; }
}
```

### Data Flow
```
beforeReadFile → ingest.cjs (attachment_rules, attachment_skills)
                       │
                  TCP/JSONL
                       │
server eventProcessor → case "file_read" → INSERT agent_chips (rule/skill)
                       │
                  SSE → workflowReducer (existing handleChipEvent)
                       │
                  agent.skills[], agent.rules[] populated
                       │
                  InspectorPanel renders chips
```

## Section 2: WorkflowState Enrichment (Client-side)

**Problem:** AgentNode type only has basic status fields. No per-agent counters for UI display.

**Solution:** Extend `AgentNode` type and `workflowReducer` to track counters from new event types.

### Files

- **Modify:** `src/shared/workflowTypes.ts`
- **Modify:** `src/shared/workflowReducer.ts`

### New fields on AgentNode

```typescript
export type AgentNode = {
  // ... existing fields ...
  toolCallCount: number;
  toolErrorCount: number;
  fileReadCount: number;
  fileEditCount: number;
  shellCommandCount: number;
  mcpCallCount: number;
  subagentCount: number;
  durationMs: number | null;
  lastFile: string | null;
};
```

Initialize to 0/null in agent creation paths within the reducer.

### Reducer changes — per case counter increments

| eventType | Counter update |
|-----------|---------------|
| `tool_start` | `agent.toolCallCount += 1` |
| `tool_done` (ok=false) | `agent.toolErrorCount += 1` |
| `file_read` | `agent.fileReadCount += 1`, `agent.lastFile = fields.file_path` |
| `file_edit` | `agent.fileEditCount += 1`, `agent.lastFile = fields.file_path` |
| `shell_start` | `agent.shellCommandCount += 1` |
| `mcp_start` | `agent.mcpCallCount += 1` |
| `subagent_start` | `agent.subagentCount += 1` (on parent agent) |
| `subagent_stop` | `agent.durationMs = fields.duration_ms` |
| `agent_stop` | `agent.durationMs = normalizeDuration` |
| `tab_file_read` | `agent.fileReadCount += 1` (treat as file_read) |
| `tab_file_edit` | `agent.fileEditCount += 1` |

Existing `updateAgent()` helper used for immutable state updates. Each case finds the agent via `conversationToAgentId` mapping (already in the reducer).

## Section 4: InspectorPanel Enrichment

**Problem:** Inspector only shows basic fields + chips. No tool/file/shell/MCP history per agent.

**Solution:** Pass `events` array to InspectorPanel, filter by agent, render categorized sections.

### Files

- **Modify:** `src/components/InspectorPanel.tsx`
- **Modify:** `src/App.tsx` (pass new prop)

### Props changes

```typescript
type Props = {
  agent: AgentNode | undefined;
  events: LogEvent[];  // NEW — full event buffer for filtering
};
```

### New sections (rendered conditionally)

**1. Tool Calls** — filter `events` for `tool_start`/`tool_done` matching agent's tool_use_ids:
```tsx
{filteredTools.length > 0 && (
  <div className="inspector__section">
    <h4 className="inspector__section-title">Tool Calls ({toolCount})</h4>
    <ul className="inspector__tool-list">
      {filteredTools.slice(-10).map(t => (
        <li key={t.id} className={`inspector__tool-item ${t.ok === 'false' ? 'inspector__tool-item--error' : ''}`}>
          <span className="inspector__tool-name">{t.toolName}</span>
          <span className="inspector__tool-status">{t.ok === 'true' ? '✓' : '✗'}</span>
          <span className="inspector__tool-duration">{formatMs(t.durationMs)}</span>
        </li>
      ))}
    </ul>
  </div>
)}
```

**2. File Operations** — filter for `file_read`/`file_edit` events, deduplicate by file_path. Show with icons.

**3. Shell Commands** — filter for `shell_start` events, show command_summary + duration + exit_hint.

**4. MCP Calls** — filter for `mcp_start` events, show tool_name + duration.

**5. Subagents** — match child agents whose `parent_agent_id === agent.id`. Show label, duration, toolCallCount, status.
